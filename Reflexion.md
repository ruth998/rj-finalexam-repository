This project was both challenging and educational because it required me to build a complete inventory and purchase order system using multiple Python files and object-oriented programming. One of the hardest parts of the project was making sure that all the different components of the system worked together correctly. Since the program is divided into several parts such as products, vendors, purchase orders, inventory management, and reports, it was sometimes difficult to ensure that data flowed correctly between them without errors. I also had to carefully manage how objects were created and used across the system, which required a strong understanding of classes and parameters.

The most difficult bug I encountered was related to the `TypeError` that occurred in both the Vendor and PurchaseOrder classes. This issue happened because the number of arguments I was passing when creating objects did not match the number of parameters defined in the class constructors. For example, I initially tried to create a Vendor object using only an ID and name, but the class required additional information such as contact details, phone number, email, and address. Similarly, my PurchaseOrder class was designed to use a list of items, but I was incorrectly passing separate values for product ID and quantity. It took me a long time to identify the mismatch between the class definitions and the object creation, but once I corrected the structure and made everything consistent, the program started working properly.

To organize my code, I separated it into multiple files based on functionality. I used a `models.py` file to define the main classes such as Product, Vendor, and PurchaseOrder. I created an `inventory_manager.py` file to handle operations like adding products, searching, and managing stock levels. The `file_manager.py` file was responsible for saving and loading data using JSON files, ensuring that information could be stored and retrieved between sessions. I also used a `reports.py` file to generate summaries such as inventory reports and low stock alerts. Finally, the main file controls the user interface and connects all the modules together through a menu system.

The save and load system works by converting objects into dictionaries using a `to_dict()` method inside each class. These dictionaries are then saved into JSON files so that the data is stored permanently. When the program is restarted, the JSON files are read, and the data is converted back into objects so the system can continue from where it left off. This method allows the program to maintain persistent data without losing information when it closes.

If I had another week to improve the project, I would focus on enhancing the user experience and adding more advanced features. I would improve error handling to make the program more stable when users enter incorrect input. I would also expand the reporting system by adding more detailed analytics such as sales trends and inventory predictions. Additionally, I would consider replacing the JSON file system with a database like SQLite to make the system more efficient and scalable. Finally, I would improve the interface to make it more user-friendly and easier to navigate for non-technical users.
